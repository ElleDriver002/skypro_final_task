Git

  • git status — показывает текущее состояние репозитория: какие файлы изменены, какие добавлены в индекс, какие не отслеживаются.
  • git add [файл] — добавляет файлы в индекс (подготавливает их к коммиту). Для добавления всех изменённых файлов можно использовать git add ..
  • git commit -m "Сообщение" — создаёт коммит с указанными изменениями и комментарием. Коммит фиксирует состояние проекта на определённый момент.
  • git log — отображает историю коммитов: список всех коммитов с их хешами, авторами, датами и сообщениями.
  • git branch [имя_ветки] — создаёт новую ветку. Без аргументов выводит список существующих веток.
  • git checkout [ветка] — переключает на указанную ветку. Для создания и переключения на новую ветку можно использовать git checkout -b [имя_ветки].
  • git merge [ветка] — объединяет указанную ветку с текущей. Используется для вливания изменений из одной ветки в другую.
  • git push — отправляет локальные коммиты в удалённый репозиторий (например, на GitHub).
  • git pull — загружает изменения из удалённого репозитория и автоматически объединяет их с текущей веткой.
  • git clone [URL] — создаёт локальную копию удалённого репозитория. Используется для начала работы над проектом.
